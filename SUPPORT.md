# Support

## Getting Help

There are several ways to get help with flowspec:

### Documentation

- **[README](./README.md)** - Getting started, installation, quick reference
- **[User Guides](./docs/guides/)** - In-depth guides for workflows and features
- **[Reference Documentation](./docs/reference/)** - Command and API reference

### Community

- **[GitHub Discussions](https://github.com/jpoley/flowspec/discussions)** - Ask questions, share ideas, discuss features
- **[Issue Tracker](https://github.com/jpoley/flowspec/issues)** - Report bugs and request features

### Issue Templates

When filing issues, please use our templates:

- **[Bug Report](.github/ISSUE_TEMPLATE/bug_report.yml)** - Report unexpected behavior
- **[Feature Request](.github/ISSUE_TEMPLATE/feature_request.yml)** - Suggest new features

### Security Issues

**Do not file public issues for security vulnerabilities.**

See [SECURITY.md](./SECURITY.md) for responsible disclosure process.

## What to Expect

### Response Times

- **Bug reports**: Triage within 7 days
- **Feature requests**: Review within 14 days
- **Security issues**: Initial response within 48 hours

### Issue Lifecycle

1. **Triage**: Maintainers review and label the issue
2. **Discussion**: Clarify requirements if needed
3. **Implementation**: Work is planned and executed
4. **Resolution**: Issue is closed with appropriate status

### Stale Issues

Issues with no activity for 60 days may be marked as stale. Stale issues are closed after an additional 7 days without activity. This helps keep the backlog manageable.
